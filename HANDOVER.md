### アプリの特徴(記事の訴求ポイント)

1. **🎲 ルーレットで即決**: ボタンひとつで家庭料理36品からランダムに1品。料理名がパラパラ切り替わってから止まる演出で、決まった感がある
2. **📝 ルーレットの中身を自分で追加できる(マイ料理)**: 「うちの塩ちゃんこ鍋」「餃子(冷凍)」のような自分の定番・テイクアウトを、主役・時間・こってり/さっぱりを選んで登録。材料や作り方は省略してもOK。内蔵36品と一緒にルーレット・気分診断の候補になる
3. **絞り込みは2つだけ**: 「主役(お肉/お魚/麺・ごはん/卵・豆腐/おまかせ)」と「調理時間(15/20/30分以内)」。考えることを最小限に
4. **🔮 3問の気分診断**: 「こってり?さっぱり?」「時間はどれくらい?」「主役は?」に答えるとおすすめベスト3。ルーレットの運任せが嫌な日はこちら
5. **決めたらそのまま作れる**: 内蔵レシピは材料(2人分)・調味料の分量・作り方つき。「材料リストをコピー」で買い物メモにもなる
6. **同じ料理が続かない工夫**: 直近3回に決めた料理はルーレットに出にくくなる
7. **結果コピーでシェア**: 「今日の夕飯は『豚の生姜焼き』に決定!🎲」をコピーして家族LINEやSNSに送れる
8. データは端末内のみ・登録不要

### 想定読者(記事の切り口)

「献立を考えるのが一番の家事」問題への、いちばんハードルが低い入口。診断・ルーレット系はSNSでシェアされやすく、ブログの「遊べる記事」枠。**姉妹アプリへの導線が本命**: じっくり派には「週の献立プランナー」(1週間まとめて計画)、冷蔵庫と相談したい人には「のこりレシピ」(食材から探す)を記事内で紹介し、3アプリの回遊を作る。

## 記事の下書き(そのまま使用・編集可)

タイトル案:
- 「『今日の夕飯どうしよう』を10秒で終わらせる無料アプリ『夕飯ルーレット』」
- 「献立を考えるのに疲れた日は、ルーレットに決めてもらおう」

---

**毎日の「夕飯どうしよう」、地味に疲れませんか。**

メニューを考えるのは、買い物や調理と並ぶ立派な家事。「夕飯ルーレット」は、その悩む時間を10秒で終わらせる無料アプリです。

**ボタンを押すだけ。あとはルーレットが決めてくれる**

🎲ボタンを押すと、定番の家庭料理36品から今日の1品がランダムに決まります。「お肉の気分」「15分以内で作れるもの」のような絞り込みも2タップ。直近に決めた料理は出にくくなるので、連日カレーみたいなことにはなりません。

**運任せが嫌な日は、3問の気分診断で**

「こってり?さっぱり?」「どれくらい時間かける?」「主役は?」——3問に答えると、今日のあなたに合うおすすめベスト3を提案してくれます。

**決まったら、そのまま作れる**

どの料理にも2人分の材料・調味料の分量・作り方がついています。「材料リストをコピー」すれば、そのまま買い物メモに。「今日の夕飯は『◯◯』に決定!」を家族のLINEに送って、帰りに買い物を頼む、なんて使い方もできます。

**ルーレットの中身は自分で増やせる**

「マイ料理」タブから、うちの定番料理やテイクアウトを自由に追加できます。主役・時間・こってり/さっぱりを選ぶだけで、材料や作り方は省略してもOK。内蔵の36品と一緒にルーレット・気分診断の候補になるので、使うほど"わが家仕様"のルーレットに育っていきます。

**じっくり決めたい日は姉妹アプリで**

- 1週間分まとめて決めたい → [週の献立プランナー](https://gryo1240.github.io/kondate/)
- 冷蔵庫にある食材から決めたい → [のこりレシピ](https://gryo1240.github.io/nokori/)

同じ36品のレシピを使った3兄弟です。気分で使い分けてください。

**インストール方法(無料・登録不要)**

- Android: [アプリを開く](https://gryo1240.github.io/yuhan/) → Chromeメニュー →「アプリをインストール」
- iPhone: Safariで開く → 共有ボタン →「ホーム画面に追加」

## アプリ側を修正したい場合

- 本体は `index.html` 1ファイル。`main` に push で GitHub Pages に自動デプロイ
- レシピデータ(`BUILTIN`・36品)は kondate / nokori と同一内容をこのファイル内に複製して持っている(単一HTML方針のため共有モジュールなし)。レシピ修正時は3アプリを揃えるかどうか判断が必要
- こってり/さっぱりの分類は `TASTE` マップ(k=こってり, s=さっぱり, n=どちらでも)。新内蔵レシピ追加時はここにも1行追加
- **ユーザー拡張(2026-07-16追加)**: マイ料理タブ(`renderMine()`)で自分の料理を追加できる(`S.custom`、id接頭辞 "c"、taste はレコード自身に持つ→`tasteOf()`)。`allRecipes()`=BUILTIN+custom がルーレット・診断の候補
- ルーレットは `spin()`: candidates() から一様ランダム。演出は setTimeout の間隔を段々広げるだけ(prefers-reduced-motion では演出なしで即結果)
- 候補除外は `candidates()`: 直近3件の履歴を除外、ただし残りが5品未満になる場合は除外しない
- 診断は `renderShindan()` 内: 主役・時間はハードフィルタ、味の好みはスコア(一致+3 / どちらでも+1.5 / 不一致+0)+乱数
- localStorageキー: `yuhan-v1`(スキーマは後方互換で custom を追加)。同期機能はなし(全データ端末内)
- sw.js のキャッシュ名は `yuhan-` プレフィックス(他アプリのキャッシュを消さないよう activate で自プレフィックスのみ削除)
